# zod-envkit - Type-Safe Environment Variable Validation and Documentation

## Overview

zod-envkit is a lightweight TypeScript library and CLI that validates environment variables at startup using Zod schemas, giving you fully typed `process.env` access and failing fast before deployment. It also generates `.env.example` files and documentation (ENV.md, ENV.json, ENV.yaml) from a single source of truth, keeping your env config, examples, and docs always in sync.

## Core Features

- **Zod-based validation** — validate `process.env` at startup with full Zod schema power (enums, coercion, URLs, etc.)
- **Full TypeScript inference** — `env.PORT` is a number, not `string | undefined`
- **Fail-fast mode** — `mustLoadEnv()` throws immediately if env is invalid, preventing runtime surprises
- **Doc generation** — auto-generate `.env.example`, `ENV.md`, `ENV.json`, or `ENV.yaml` from `env.meta.json`
- **CI/CD validation** — `zod-envkit check` exits with code 1 on missing vars; `--strict` mode also fails on unknown variables
- **Environment inspection** — `zod-envkit show` displays current env state with automatic secret masking
- **Multi-dotenv loading** — load multiple `.env*` files with priority ordering
- **Bootstrap tooling** — `zod-envkit init` generates metadata from existing `.env.example` or vice versa

## Quick Start Guide

1. Install: `npm install zod-envkit` (or yarn/pnpm)
2. Create your env schema file:
   ```typescript
   import "dotenv/config";
   import { z } from "zod";
   import { mustLoadEnv } from "zod-envkit";
   
   const EnvSchema = z.object({
     NODE_ENV: z.enum(["development", "test", "production"]),
     PORT: z.coerce.number().int().min(1).max(65535),
     DATABASE_URL: z.string().url(),
   });
   
   export const env = mustLoadEnv(EnvSchema);
   ```
3. Create `env.meta.json` for CLI features (or bootstrap with `npx zod-envkit init`)
4. Generate docs and `.env.example`: `npx zod-envkit generate`
5. Add CI check: `npx zod-envkit check --strict`

## Use Cases

- **TypeScript projects** — replace unsafe `process.env` access with validated, typed environment variables
- **CI/CD pipelines** — catch missing or unknown env vars before deployment with `check --strict`
- **Team onboarding** — auto-generate up-to-date `.env.example` and documentation from a single source
- **Microservices** — validate env contracts at startup across multiple services
- **Debugging deployments** — inspect current env state with secret masking via `show` command

## Notes & Considerations

zod-envkit is designed to complement dotenv, not replace it — use dotenv for loading `.env` files and zod-envkit for validation and typing. The library and CLI are independent but complementary: you can use just the runtime validation without the CLI, or just the CLI for doc generation. The `env.meta.json` file is required for CLI commands but not for library usage. No framework coupling — works anywhere Node.js runs.
