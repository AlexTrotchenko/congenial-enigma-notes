# Sablier - Scale-to-Zero for Containers on Demand

## Overview

Sablier is a free, open-source tool that starts workloads on demand and automatically stops them after a period of inactivity. It integrates with reverse proxy plugins (Traefik, Caddy, Nginx, Envoy, etc.) to intercept incoming requests, wake up sleeping containers, and display a waiting page until they're ready. Perfect for Raspberry Pi setups, dev/QA environments, or any situation where keeping containers running 24/7 wastes resources.

## Core Features

- **On-demand container start** — workloads start only when requests arrive
- **Automatic shutdown** — containers stop after configurable inactivity period
- **Waiting page** — customizable UI shown while containers are starting
- **Multiple providers** — Docker, Docker Swarm, Podman, Kubernetes
- **Reverse proxy plugins** — Traefik, Caddy, Nginx, Envoy, Istio, Apache APISIX
- **Session management** — configurable session duration and expiration
- **Health check awareness** — waits for containers to be healthy before routing
- **Custom themes** — multiple built-in themes for the waiting page
- **Helm chart** — production-ready Kubernetes deployment

## Supported Providers

| Provider | Features |
|----------|----------|
| Docker | Start/stop containers, Docker Compose compatible |
| Docker Swarm | Scale services 0↔N, Docker Stack compatible |
| Podman | Daemonless containers, rootless support |
| Kubernetes | Scale Deployments/StatefulSets, in-cluster & out-of-cluster |

## Reverse Proxy Integrations

- **Traefik** — Native middleware plugin
- **Caddy** — Native Caddy module (xcaddy)
- **Nginx** — Proxy-WASM plugin
- **Envoy** — Proxy-WASM plugin
- **Istio** — EnvoyFilter with Proxy-WASM
- **Apache APISIX** — Proxy-WASM plugin

## Quick Start Guide

1. Run a container with Sablier labels:
   ```bash
   docker run -d --name myapp \
     --label sablier.enable=true \
     --label sablier.group=demo \
     myimage:latest
   ```

2. Start Sablier:
   ```bash
   docker run -p 10000:10000 \
     -v /var/run/docker.sock:/var/run/docker.sock \
     sablierapp/sablier:1.11.1 \
     start --provider.name=docker
   ```

3. Request a session (or configure reverse proxy):
   ```bash
   curl http://localhost:10000/api/strategies/blocking?group=demo&session_duration=20s
   ```

4. Container starts automatically, stops after 20s of inactivity

## Configuration

```yaml
# sablier.yaml
provider:
  name: docker  # docker, swarm, kubernetes, podman
server:
  port: 10000
sessions:
  default-duration: 5m
  expiration-interval: 20s
strategy:
  dynamic:
    default-theme: hacker-terminal
    default-refresh-frequency: 5s
  blocking:
    default-timeout: 1m
```

## Use Cases

- **Home lab / Raspberry Pi** — don't overload limited resources with idle containers

- **Microservices** — scale rarely-used services to zero between requests
- **Cost optimization** — reduce cloud bills by stopping unused workloads
- **Demo environments** — start demos on-demand without keeping them running

## Notes & Considerations

⚠️ Always use health checks with containers you want to scale to zero — without them, Sablier can't distinguish between "started" and "ready to receive requests." The waiting page themes include "hacker-terminal" (default), and you can add custom themes. Similar concept to AWS Lambda cold starts, but for any containerized workload. Maintained by a single developer — consider starring/contributing if you find it useful.
